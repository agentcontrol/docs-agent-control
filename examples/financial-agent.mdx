## Overview

This use case focuses on financial-agent workflows (e.g., onboarding, reporting, and transaction review). Agent Control helps enforce SOX- and PII-related policies at runtime.

## Typical risks addressed

- **PII leakage** in customer records
- **Prompt injection** in document ingestion
- **Unapproved tool usage** for sensitive actions
- **Auditability** with policy-based controls

## Suggested control set

### 1) Block PII in outputs

```json
{
  "name": "block-pii-output",
  "execution": "server",
  "scope": { "step_types": ["llm"], "stages": ["post"] },
  "selector": { "path": "output" },
  "evaluator": {
    "name": "regex",
    "config": { "pattern": "\\b\\d{3}-\\d{2}-\\d{4}\\b" }
  },
  "action": { "decision": "deny" }
}
```

### 2) Prevent prompt injection in inputs

```json
{
  "name": "block-injection-input",
  "execution": "server",
  "scope": { "step_types": ["llm"], "stages": ["pre"] },
  "selector": { "path": "input" },
  "evaluator": {
    "name": "regex",
    "config": { "pattern": "(?i)ignore previous instructions|system prompt" }
  },
  "action": { "decision": "deny" }
}
```

### 3) Restrict sensitive tools

```json
{
  "name": "limit-sensitive-tools",
  "execution": "server",
  "scope": {
    "step_types": ["tool"],
    "step_names": ["transfer_funds", "update_account", "close_account"],
    "stages": ["pre"]
  },
  "selector": { "path": "name" },
  "evaluator": { "name": "list", "config": { "values": ["transfer_funds"] } },
  "action": { "decision": "deny" }
}
```
